Etapas

EDA

Analise exploratoria com pandas e matplotlib. Distribuicao de features, correlacoes e outliers no catalogo musical.

KNN Model

NearestNeighbors do scikit-learn treinado nas features audio do Spotify. Distancia cosseno para similaridade entre faixas.

MLflow

Rastreabilidade de experimentos: hiperparametros, metricas e artefatos versionados. Comparacao entre runs no UI.

Gradio

Interface web para selecionar uma musica e receber recomendacoes similares em tempo real com visualizacao.

Codigo destaque

from sklearn.neighbors import NearestNeighbors
import mlflow
import pandas as pd

def train_recommender(tracks: pd.DataFrame, n_neighbors: int = 10):
    """Treina KNN nas features audio e loga no MLflow."""
    features = tracks[["danceability", "energy", "valence", "tempo"]]

    with mlflow.start_run():
        model = NearestNeighbors(
            n_neighbors=n_neighbors,
            metric="cosine",
        )
        model.fit(features)

        mlflow.log_param("n_neighbors", n_neighbors)
        mlflow.log_param("metric", "cosine")
        mlflow.log_metric("n_tracks", len(tracks))
        mlflow.sklearn.log_model(model, "knn_recommender")

    return model

def recommend(model, tracks: pd.DataFrame, track_idx: int) -> pd.DataFrame:
    """Retorna as faixas mais similares por distancia cosseno."""
    features = tracks[["danceability", "energy", "valence", "tempo"]]
    distances, indices = model.kneighbors(features.iloc[[track_idx]])
    return tracks.iloc[indices[0]]

O que aprendi

  • KNN para recomendacao — Distancia cosseno em features normalizadas gera recomendacoes coerentes sem treino supervisionado.
  • MLflow experiment tracking — Cada run versionada com parametros, metricas e modelo serializado. Comparacao visual entre experimentos.
  • Gradio como prototipagem rapida — Interface funcional com dropdown e tabela de resultados em poucas linhas.
  • EDA orientada a dominio — Entender distribuicao das features audio antes de escolher metrica e numero de vizinhos.
