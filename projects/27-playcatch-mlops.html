Etapas

Data + Train

Ingestao de dados, feature engineering e treino do modelo KNN. Cada run logada no MLflow com parametros e artefatos.

Evaluate

Avaliacao automatica com metricas de precisao e recall. Gate de qualidade: modelo so promove se passar threshold.

Deploy

API FastAPI com validacao Pydantic servida em container Docker non-root. Healthcheck e rate limiting integrados.

Monitor

Deteccao de data drift por comparacao de distribuicoes. Alerta automatico e trigger de re-treino quando drift detectado.

Codigo destaque

from fastapi import FastAPI, HTTPException
from pydantic import BaseModel
import mlflow.sklearn
import numpy as np

app = FastAPI()
model = mlflow.sklearn.load_model("models:/knn_recommender/Production")

class RecommendRequest(BaseModel):
    track_features: list[float]
    n_results: int = 5

@app.post("/recommend")
def recommend(req: RecommendRequest):
    features = np.array(req.track_features).reshape(1, -1)
    distances, indices = model.kneighbors(
        features, n_neighbors=req.n_results,
    )
    return {
        "indices": indices[0].tolist(),
        "distances": distances[0].tolist(),
    }

# Drift detection
def detect_drift(reference: np.ndarray, current: np.ndarray, threshold: float = 0.05) -> bool:
    """KS-test por feature: drift se p-value abaixo do threshold."""
    from scipy.stats import ks_2samp
    for col in range(reference.shape[1]):
        stat, pvalue = ks_2samp(reference[:, col], current[:, col])
        if pvalue < threshold:
            return True
    return False

Seguranca

  • Container non-root — Dockerfile com USER nao-root, cap_drop ALL e filesystem read-only.
  • Rate limiting — Limite por IP na API para prevenir abuso de recursos.
  • Structured logging — Logs JSON com request_id para rastreabilidade ponta-a-ponta.
  • Drift detection — KS-test automatico detecta degradacao antes de impactar usuarios.

O que aprendi

  • Pipeline MLOps completo — Do treino ao monitoramento em producao, cada etapa automatizada e versionada.
  • Model registry com MLflow — Promocao de modelo por stages (Staging, Production) com rollback trivial.
  • Container hardening — Non-root, read-only, healthcheck: seguranca sem complexidade.
  • Drift como trigger — Monitoramento proativo que dispara re-treino automaticamente.
